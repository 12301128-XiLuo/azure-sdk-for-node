beginCreate(resourceGroupName, profileName, endpointName, endpoint, optionsopt, optionalCallbackopt) → {function|Promise}

Creates a new CDN endpoint with the specified endpoint name under the
specified subscription, resource group and profile.

Parameters:
Name Type Attributes Description
resourceGroupName string

Name of the Resource group within the
Azure subscription.
profileName string

Name of the CDN profile which is unique within
the resource group.
endpointName string

Name of the endpoint under the profile which is
unique globally.
endpoint object

Endpoint properties

Properties
Name Type Attributes Description
originHostHeader string <optional>

The host header CDN sends along
with content requests to origin. The default value is the host name of the
origin.
originPath string <optional>

The path used when CDN sends request
to origin.
contentTypesToCompress array <optional>

List of content types on
which compression applies. The value should be a valid MIME type.
isCompressionEnabled boolean <optional>

Indicates whether content
compression is enabled on CDN. Default value is false. If compression is
enabled, content will be served as compressed if user requests for a
compressed version. Content won't be compressed on CDN when requested
content is smaller than 1 byte or larger than 1 MB.
isHttpAllowed boolean <optional>

Indicates whether HTTP traffic is
allowed on the endpoint. Default value is true. At least one protocol (HTTP
or HTTPS) must be allowed.
isHttpsAllowed boolean <optional>

Indicates whether HTTPS traffic
is allowed on the endpoint. Default value is true. At least one protocol
(HTTP or HTTPS) must be allowed.
queryStringCachingBehavior string <optional>

Defines the query
string caching behavior. Possible values include: 'IgnoreQueryString',
'BypassCaching', 'UseQueryString', 'NotSet'
optimizationType string <optional>

Customer can specify what
scenario they want this CDN endpoint to optimize, e.g. Download, Media
services. With this information we can apply scenario driven optimization.
geoFilters array <optional>

List of rules defining user geo access
within a CDN endpoint. Each geo filter defines an acess rule to a specified
path or content, e.g. block APAC for path /pictures/
origins array

The source of the content being delivered
via CDN.
location string

Resource location.
tags object <optional>

Resource tags.
options object <optional>

Optional Parameters.

Properties
Name Type Attributes Description
customHeaders object <optional>

Headers that will be added to the
request
optionalCallback function <optional>

The optional callback.
